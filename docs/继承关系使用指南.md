# 继承关系使用指南

本文档说明如何选择和使用不同的继承相关关系类型。

---

## 三种"继承"关系的区别

### 1. `inherits_from` - OOP继承

**含义**：代码层面的继承，子类获得父类的属性和方法

**使用场景**：编程语言中的类继承

**示例**：
```yaml
---
# 在 ArrayList.md 中
inherits_from: "[[AbstractList]]"
---
```

```java
class ArrayList extends AbstractList {
    // ArrayList 继承了 AbstractList 的所有方法
}
```

---

### 2. `subclass_of` - 概念分类

**含义**：概念层级中的"是一种"关系，表示分类学上的从属

**使用场景**：知识分类、生物分类、概念层次

**示例**：
```yaml
---
# 在 狗.md 中
subclass_of: "[[哺乳动物]]"
---
```

**概念层级**：
```
动物
 └─ 脊椎动物
     └─ 哺乳动物
         └─ 犬科
             └─ 狗
```

---

### 3. `extends` - 扩展延伸

**含义**：在原有基础上扩展新功能或理论

**使用场景**：理论扩展、功能增强

**示例**：
```yaml
---
# 在 广义相对论.md 中
extends: "[[狭义相对论]]"
---
```

广义相对论在狭义相对论基础上，增加了引力和加速度的处理。

---

## 关系对比表

| 关系类型 | 领域 | 强调点 | 方向性 | 示例 |
|---------|------|--------|--------|------|
| `inherits_from` | 编程 | 代码复用、方法获得 | 单向 | ArrayList inherits_from List |
| `subclass_of` | 分类学 | "是一种"、概念包含 | 单向 | 狗 subclass_of 哺乳动物 |
| `extends` | 理论/功能 | 扩展、延伸 | 单向 | TypeScript extends JavaScript |
| `derived_from` | 概念起源 | 来源、派生 | 单向 | 信息论 derived_from 熵 |
| `instance_of` | 实例 | 实例与类的关系 | 单向 | Fido instance_of Dog |

---

## 使用场景示例

### 场景1：编程语言的类继承

```yaml
---
# ArrayList.md
inherits_from: "[[AbstractList]]"
---

# ArrayList

Java 中的动态数组实现，继承自 AbstractList。
```

**关系链**：
```
Collection (接口)
 └─ List (接口)
     └─ AbstractList (抽象类)
         └─ ArrayList (实现类)  ← inherits_from
```

---

### 场景2：生物分类学

```yaml
---
# 黄金猎犬.md
subclass_of: "[[犬科]]"
instance_of: "[[犬种]]"
---

# 黄金猎犬

一种温顺友善的大型犬。
```

**关系链**：
```
哺乳动物
 └─ 食肉目
     └─ 犬科           ← subclass_of
         └─ 家犬
             └─ 黄金猎犬  ← 某只具体的黄金猎犬 instance_of 黄金猎犬
```

---

### 场景3：理论发展

```yaml
---
# 量子场论.md
extends: "[[量子力学]]"
extends: "[[相对论]]"
---

# 量子场论

结合了量子力学和相对论的理论框架。
```

**关系链**：
```
经典力学
 ├─ 量子力学
 │   └─ 量子场论  ← extends
 └─ 相对论
     └─ 量子场论  ← extends
```

---

## 最佳实践

### ✅ 推荐

**编程相关笔记**：使用 `inherits_from`
```yaml
---
# Dog.java
inherits_from: "[[Animal]]"
---
```

**概念分类笔记**：使用 `subclass_of`
```yaml
---
# 红色.md
subclass_of: "[[颜色]]"
---
```

**理论扩展笔记**：使用 `extends`
```yaml
---
# 深度学习.md
extends: "[[机器学习]]"
---
```

---

### ⚠️ 常见混淆

**错误示例1**：混淆 subclass_of 和 instance_of
```yaml
# ❌ 错误
---
# Fido.md (一只具体的狗)
subclass_of: "[[Dog]]"  # 错！Fido 不是 Dog 的子类
---

# ✅ 正确
---
# Fido.md
instance_of: "[[Dog]]"  # Fido 是 Dog 的一个实例
---
```

**错误示例2**：混淆 inherits_from 和 extends
```yaml
# ❌ 不精确
---
# TypeScript.md
inherits_from: "[[JavaScript]]"  # 不太准确，TS不是继承JS的类
---

# ✅ 更准确
---
# TypeScript.md
extends: "[[JavaScript]]"  # TS 扩展了 JS 的功能
---
```

## 完整示例对比

### 编程场景
```yaml
---
# Dog.java
inherits_from: "[[Animal.java]]"
---

class Dog extends Animal {
    void bark() { }
}
```

### 概念分类场景
```yaml
---
# 狗.md（概念）
subclass_of: "[[哺乳动物]]"
---

# 狗

犬科哺乳动物，人类最早驯化的动物之一。
```

### 理论场景
```yaml
---
# 广义相对论.md
extends: "[[狭义相对论]]"
---

# 广义相对论

爱因斯坦在狭义相对论基础上，将引力纳入时空框架。
```

---

## 总结

- **inherits_from**：代码继承 → 用于编程
- **subclass_of**：概念分类 → 用于知识组织
- **extends**：扩展延伸 → 用于理论发展

选择哪个取决于你的笔记领域和语义需求！
